Factor Analysis

Factor Analysis is a statistical technique for identifying patterns in observed variables and explaining connections between them in terms of underlying, unobserved variables called as factors.Factor analysis assumes that the observed variables are influenced by a small number of unobserved variables, or factors, which are responsible for the observed correlations among the variables.
These factors are extracted from the data using statistical methods, and the amount of variance each factor explains is called its eigenvalue.

we are performing factor analysis on our data frame fa_topsongs consisting of top songs data with their audio features. We are excluding the first four columns from the data frame as they contain non-numeric data.

We want to extract two factors and thus we have set nfactors to 2. We have rotate set to varimax because rotation simplifies the factor structure and makes it more interpretable to analyze. And the factoring method used is principal axis factoring

#Factor Analysis
fa_topsongs <- data.frame(topsongs.df[-c(1,1:4)])
fa_topsongs <- fa(fa_topsongs, nfactors=2, rotate="varimax", fm="pa")

In factor analysis, loading is a measure of the relationship between the observed variables and the underlying factors. It represents the correlation between the variables and the factors.They are expressed as numbers ranging from -1 to 1, with higher absolute values indicating stronger relationships between the variables and the factors.If a variable has a high loading on a particular factor, it is likely that the variable is measuring the same underlying construct as the other variables that also load highly on that factor.

Principal Component Analysis (PCA)

Principal Component Analysis is used to analyze and reduce the dimensionality of a large dataset.
In PCA, a set of variables that are highly correlated with each other are transformed into a new set of uncorrelated variables, called principal components. These principal components are linear combinations of the original variables and are ranked in order of importance, with the first principal component capturing the largest amount of variation in the data, the second component capturing the second-largest amount of variation, and so on.

PCA involves finding the eigenvectors and eigenvalues of the covariance matrix of the original dataset. The eigenvectors represent the direction of maximum variance in the data and form the new set of uncorrelated variables (i.e., the principal components). The eigenvalues represent the amount of variance in the data that is explained by each principal component.

#Principal Component Analysis
pca_topsongs <- data.frame(topsongs.df[-c(1,1:4)])
pca_topsongs <- principal(pca_topsongs[-c(11,12)], nfactors=1, score=TRUE)

Loadings refer to the values that indicate the strength of the relationship between the original variables and the principal components.

After performing PCA, you get a set of principal components that are a linear combination of the original variables. The loadings represent the weights of each variable in the linear combination of the principal component.

K-means Clustering

K-means clustering is a popular unsupervised machine learning technique used for partitioning a dataset into k distinct, non-overlapping clusters based on their attributes. It aims to minimize the sum of squared distances between data points and their assigned cluster centers. We perform k-means clustering using the kmeans() function, which takes as input a dataset and the desired number of clusters (k). The function then returns the cluster assignments and centroids.

We standardized the values of columns “PA1” and “PA2” using the scale() function and stored them in a new data frame called “df.norm”.This step is necessary for k-means clustering because the algorithm works based on the distance between points, and standardizing the values of the variables ensures that they are on the same scale and have equal influence in the distance calculation.

#k-means clustering 
df <- data.frame(topsongs.df[c("title","PA1", "PA2")])
df.norm <- sapply(df[,-1], scale)
row.names(df.norm) <- row.names(df)

We use the elbow method to calculate the desired number of clusters.
The elbow method is a heuristic method used to determine the optimal number of clusters to use in k-means clustering.

#elbow method to determine number of clusters
wss <- sapply(1:10, function(k) {
  kmeans(df.norm, k, nstart = 50)$tot.withinss
})
plot(1:10, wss, type="b", pch = 19, frame = FALSE,
     xlab="Number of clusters K",
     ylab="Total Within-Clusters Sum of Squares")

#The number of clusters = 4

The optimal number of clusters to use is 4

We apply the k-means clustering to our normalized data. The output km contains information about the clusters such as cluster membership, cluster centers, and within-cluster sum of squares.
We store the details of cluster membership in the dataframe km_list and add it to our main dataframe

Hierarchical Clustering

Hierarchical clustering is a popular method of clustering analysis that involves creating a hierarchy of clusters, often depicted as a dendrogram, through a series of successive partitions of a dataset. In hierarchical clustering, the data points are initially considered as individual clusters, and then pairs of clusters are merged in a way that minimizes a similarity or distance metric until all data points belong to the same cluster.

Hierarchical clustering can be visualized through a dendrogram, which displays the clusters and their merging sequence as a tree-like diagram. This makes it easier to interpret the results and to determine the optimal number of clusters. The choice of the distance metric used to compute the similarity between pairs of data points here is euclidean distance.

#hierarchical clustering
df <- data.frame(topsongs.df[c("title","PA1", "PA2")])
df.norm <- sapply(df[,-1], scale)
row.names(df.norm) <- row.names(df)
d.norm <- dist(df.norm, method = "euclidean")
hc <- hclust(d.norm)
#plotting the clusters
avg_dend_obj <- as.dendrogram(hc)
avg_col_dend <- color_branches(avg_dend_obj, h = 4)
plot(avg_col_dend)

n_clusters=4
heir.cluster <- cutree(hc, k = n_clusters)# 4 is the number of clusters
# Adding the clusters to the original dataframe
heir.cluster_list <- data.frame(heir.cluster)
topsongs.df <- cbind.data.frame(topsongs.df,heir.cluster_list)
